# GemDeck プロジェクト開発指示書 (instructions.md)

このドキュメントは、プロジェクト「GemDeck」におけるAIアシスタントの開発指針と技術仕様をまとめたものです。

## 1. 基本行動指針
- **言語**: 全ての対話、ドキュメント、コードコメントは**日本語**で行います。
- **品質**: 既存の機能を壊さない（デグレードさせない）ことを最優先に考慮し、修正後は影響範囲を確認します。
- **タスクルーティング**: ユーザーからのコードに関する指示には、まず `/route_task` ワークフローに従って、自身（Gemini）で対応するか Claude Code に任せるかを判定します。
- **ナレッジベース**: `.agent` フォルダ内のすべてのドキュメント、ワークフロー、および指示書は、プロジェクトのマスターナレッジとして扱います。起動時やタスク実行時にこれらを読み込み、常に最新のコンテキストに基づいた対応を行ってください。

## 2. プロジェクト概要
- 仕様は、`specifications.md`を参照してください。
- 開発はAntigravity上で行うため、各種設定や操作についてはAntigravityの仕様に従ってください。
- 私たち開発チームの構成については、[team.md](./team.md) を参照してください。

## 7. コード管理(Git)
- **リポジトリ**
    - https://github.com/cellsica/gemdeck

- **ブランチ構成**:
    - `main`: リリース用ブランチ。
    - `develop`: 開発のベースブランチ。削除せずに維持します。
    - `feature/xxx` または `temp/xxx`: 仮実装（タスク単位）用ブランチ。
- **ワークフロー**:
    1.  `develop` から作業用の仮実装ブランチを作成して着手します。
    2.  修正完了後、ユーザーの動作確認を得てから `develop` へマージします。
    3.  マージ後の仮実装ブランチは削除します。
    4.  `main` へのマージは、機能が一定数揃った段階でユーザーの指示に基づいて行います。

- **コミットメッセージ**: 変更内容が簡潔にわかる日本語で記述します。
